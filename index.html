How these systems actually work under the hood.

Plant Diagnosis Flow

1 User captures / uploads image
2 Compress & Base64 encode (worker thread)
3 POST → Plant.id identification API
4 Parse JSON → PlantData model
5 Results screen + BloomAI care advice

Data Architecture

↑ Android client (Activities + Fragments)
⇄ OkHttp REST layer
AI Plant.id · OpenAI · OpenWeather
DB Firebase + local storage (garden, reminders)
⌂ Home dashboard & AlarmManager reminders

Engineering Challenges

  • Keeping network calls and image encoding off the main thread to avoid ANRs
  • Defensive JSON parsing with fallbacks so a malformed API response never crashes a fragment
  • Scheduling reliable watering reminders via AlarmManager that survive reboots (BootReceiver)
  • Managing state across many fragments (Home, Garden, Camera, Weather, AI, Problems)

Key Learnings

  • Android architecture & fragment lifecycle
  • Integrating multiple third-party REST APIs
  • Designing AI-powered product flows
  • Mobile UX and state management

Slayer HUD Flow

1 Boss spawn detection
2 Entity tracking (per-player, by UUID)
3 SlayerManager collects SlayerData
4 HUD rendering pipeline
5 Real-time per-tick updates

Engineering Challenges

  • Reverse-engineering game mechanics to detect boss spawns reliably
  • Per-player state isolated with a thread-safe ConcurrentHashMap<UUID, …>
  • Real-time event handling without harming frame rate
  • Modular features that can be toggled independently

Key Learnings

  • Event-driven architecture in practice
  • Kotlin + Java interop on the JVM
  • Performance-aware rendering
  • Organizing a larger codebase into modules

Processing Pipeline

1 Texture pack input
2 Recursive folder traversal (os.walk)
3 Asset detection by name / folder
4 Hue-map lookup per category
5 HSV transform → export variant

Recoloring Logic

1 Load texture (RGBA)
2 Category detection (item / block / entity / GUI)
3 Per-pixel RGB→HSV, shift hue, HSV→RGB
4 Preserve alpha; lift near-black so hue shows
5 Write recolored output

Engineering Challenges

  • Preserving texture shading and transparency during recoloring
  • Handling many asset categories with separate hue maps & gradient rules
  • Processing hundreds of files efficiently in one pass
  • Making the rules configurable rather than hard-coded per file

Key Learnings

  • Automation & scalable script design
  • Color theory: HSV manipulation algorithms
  • File-system operations at scale
  • Designing for configurability
